JavaScript in Relation to HTML and CSS

HTML is like the actor in a movie. Before anything happens, they are just there, themselves, not doing a whole lot. CSS is like the set, the props, the costumes, all the things that give the scene some presentation. JavaScript is what makes everything happen after the director calls action.

Control Flow, Loops

Control flow is the order that things are processed/executed.
Think, driving your car. First you unlock it. Open the door. Get in. Close the door. Start the car. Drive away.

But maybe the car wasn't locked?
Then the control flow would be 'if car-is-locked then unlock-car'. Then it would move on.
This alteration to the control flow with a 'if' statement is very common in everyday life and JavaScript. We don't love when we try to unlock an unlocked car, same as we don't want a device to process information that doesn't need to be processed or we don't want processed. Like if the wrong password was used, we don't want them to be able to login anyway.
In JavaScript it might look something like this:

if (carIsLocked) { unlockCar() }

Loops can be a couple of different things. First are when something gets repeated until some parameter is met. Think about combing/brushing hair. You repeat the action over and over until the 'parameter' of having tidy hair is met. Or you might do up a button, then move on to the next button, so on, so forth until all your buttons are done up.
In JavaScript it might look like this:

for (let i=0; i < 10; i++) { brushHair() }
This loop would start with a variable i equal to 0.
It would check if i < 10.
If it is then it would do whats inside the {} brackets.
Then it adds 1 to i. So i is now 1.
Then it repeats by checking if i < 10.

The second way loops can be used would be to do something to a list of things. Say you walk into a room, theres five people it, you might go around the room and say hey, give a head nod or highfive to each person.

for (let person of listOfPeople) { highFive(person) }

This will go through the list of people in the array listOfPeople and do whatever highFive() is to each person in the array.
This type of for loop is useful when listOfPeople is an array, not an object. Which I'll explain later on.

What is the DOM?

The DOM is what your browser creates, based of the HTML document that is the basis of a webpage.
Your browser looks at the HTML document, and builds the DOM in a tree like structure.
Heres an example of part my blogs home pages HTML document.

<!DOCTYPE html> <html lang="en"> <head id="home"> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My Learning Journey</title> <link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Open+Sans"> <link rel="stylesheet" href="styles/main.css"> <script src="config.js"></script> </head> <body> </body> </html>
As a most basic explanation, it would start the trunk of the tree at <html>.
Then at <head> there would be a branch. Everything until </head> would be on that branch.
Then <body> would be a new branch that finishes when we reach </body>.
If you notice <head> has an 'id' called "home". This is a unique identifier for the <head> 'branch'.
If you wanted to do something with the <head> you could use that "home" id to find the branch and interact with it in your JavaScript file.
On my home page I've used it so that I can add a clickable 'Home' button on my blogs that take me back to the 'home' id on my main page and orient the users view to the top of the page.

You could do just about anything you can imagine to your webpage. When a browser builds the DOM from the HTML document and styles it with the CSS document it can use a whole range of languages to interact with the DOM. The DOM is quite universal, JavaScript, Python, Java, etc. can interact with the DOM.

Arrays Vs. Objects

Arrays and objects are both special kinds of 'variables'.
Arrays have information in them in a structured order. Each 'thing' in an array is called an 'element' of that array.
Heres an example of an array you might use:

let listOfPeople = ['Jackson', 'David', 'Kate', 'Jess']
listOfPeople is now an array with those names stored as elements in it.
'Jackson' is considered the 0 element, 'David' the 1 element, so on so forth.
In your scripts, if you used listOfPeople[0] your script would process that as 'Jackson'.
We should also look at the earlier example:
for (let person of listOfPeople) { highFive(person) }
This would 'loop' through the array 'listOfPeople' from element 0 to element 3. On each loop the function 'highFive' is called. The first time with the variable 'person' equal to 'Jackson', then on the second loop 'person' equals 'David', then 'Kate', then 'Jess'.

Objects are similar except they arn't 'ordered elements' they have whats called 'key-value' pairs.
These are more useful for things that need additional information. Heres an example:

let family = { jackson: { age: 15, 'birth-month': 'august' }, david: { age: 12, 'birth-month': 'february' }, kate: { age: 18, 'birth-month': 'september' }, jess: { age: 15, 'birth-month': 'august' }, }
This makes the object 'family' which contains the keys jackson, david, kate, and jess.
Each of these keys have values that are actually also objects!
Each of the family members has a key 'age' with a value which is their age and a key 'birth-month' with the value that is their birth month.
To use the information stored in objects we would do something like:
family.jackson.age
Your JavaScript would interpret that 'variable' as 15.

So arrays are great for ordered lists of information, and objects are great for storing information with multiple 'attributes' like age and birth month.
While they are kind of similar, their uses differ, and how you use the information in them differs aswell.

What are functions?

Functions are a piece of code that can have just about anything in.
You can 'feed' 'parameters' to functions like this:

function nameYourFunctionHere(parameterOne, parameterTwo) { Your function executes code in here, you can use parameterOne and parameterTwo aswell! }
This is great for situations where you want to do the same code in more than one place in your script. You can code it once and just call the function when/where you want to.
