Classes Vs. Ids - Best Practices

Classes and ID's are both used in HTML to link to specific formatting in your CSS file. These can include alignment, fonts, colouring, sizing etc.
However, classes are generally used for things that you will have in multiple places on your webpage.
Such as:

  • Sub Headings
  • Paragraphs you want formattted the same/similar
An example from my pages would be:
<div class="blog-preview"> <h3><a href="blog/javascript-dom.html">What I learnt this week! - 7 April 2024</a></h3> <p>Looking at JS, control flows, loops, the "DOM", arrays, objects, and functions.</p> </div>

This class="blog-preview" lets me do similar formatting to all my blog previews on my home page.
It links to my .css file where I have:
.blog-preview { width: 100%-40px; text-align: left; border-top: 2px solid #000000; padding-top: 20px; margin-left: 20px; margin-right: 20px; }
This allows me to format all my blog-previews in one place in my files, and its then applied to multiple places on the webpage.

These classes can let your webpages more easily adapt to things like smaller screens and display alterations for different viewers.
This is thanks to the fact you can setup one bit of code to alter the class when screen size is different. And it will be applied to all those subheadings or paragraphs.
You might use IDs for things like your webpages header that includes your title, logo, etc.
These elements are 'unique' on your webpage. They are displayed in one place with one specific format/style.
I'm still not 100% sure what I want my title to be. So I used an ID to pull in my title name from a script variable.

<h1 id="blog-title"></h1> <script> document.getElementById("blog-title").innerText = blogTitle; </script>

That id="blog-title" is a unique element which my script can fill in with the variable "blogTitle" from my config.js file:

let blogTitle = "My Learning Journey";

Both ID's and classes should be descriptive so you can easily remember what they do. This is especially important for classes where you want to use it repeatedly throughout your webpage for font coloring, font selection etc.
